# Running a Bevy App in WebAssembly

This example shows how to compile and run a Bevy app in the browser using WebAssembly (WASM).

***Note:** The Bevy version used in this example is `0.18`.*

---

## Step 1: Add the WebAssembly Target

To run your Bevy app in the browser, you first need to tell Rust to compile for WebAssembly. Run:

```sh
rustup target add wasm32-unknown-unknown
```

This adds the `wasm32-unknown-unknown` target to your Rust toolchain, which is the format that browsers can understand and run.

---

## Step 2: Install the Required Tools

You'll need two tools. Install them with:

```sh
cargo install wasm-bindgen-cli
cargo install basic-http-server
```

- **wasm-bindgen-cli** generates the JavaScript glue code that lets the browser load and run your `.wasm` file.
- **basic-http-server** serves your app locally. Browsers don't allow WASM to run from local file paths (`file://`), so you need to serve it over HTTP.

---

## Step 3: Create a New Rust Project

```sh
cargo new bevy_wasm_app
cd bevy_wasm_app
```

---

## Step 4: Add Bevy as a Dependency

Open `Cargo.toml` and add Bevy under `[dependencies]`:

```toml
[dependencies]
bevy = "0.18"
```

---

## Step 5: Configure for WebAssembly

Some browser APIs need special flags to work correctly. Create a new file at `.cargo/config.toml` inside your project folder and add:

```toml
[target.wasm32-unknown-unknown]
rustflags = ["--cfg=web_sys_unstable_apis"]
```

This enables browser-compatible APIs that Bevy relies on when running in WASM.

---

## Step 6: Build for WebAssembly

Compile your app to WebAssembly:

```sh
cargo build --target wasm32-unknown-unknown --release
```

This creates a `.wasm` file inside `target/wasm32-unknown-unknown/release/`.

---

## Step 7: Generate JavaScript Bindings

Use `wasm-bindgen` to create the JavaScript wrapper that the browser needs to load your app:

```sh
wasm-bindgen --out-dir ./out --target web ./target/wasm32-unknown-unknown/release/bevy_wasm_app.wasm
```

This creates an `out/` folder containing the `.wasm` file and the generated `.js` file.

---

## Step 8: Create an index.html

The browser needs an HTML file to load your app. Create `out/index.html` with the following:

```html
<!DOCTYPE html>
<html>
  <head>
    <meta charset="utf-8" />
    <title>Bevy WASM</title>
    <style>
      body { margin: 0; background: black; }
      canvas { display: block; }
    </style>
  </head>
  <body>
    <script type="module">
      import init from './bevy_wasm_app.js';
      init();
    </script>
  </body>
</html>
```

---

## Step 9: Serve and Run

Start a local HTTP server pointing at the `out/` folder:

```sh
basic-http-server ./out
```

Then open `http://localhost:4000` in your browser. You should see your Bevy app running!
